程序 Workflow 中文说明

这份文档用最简单的方式说明：
我们现在这个程序，从用户输入到最后输出，完整流程是什么。


====================
1. 用户输入
====================

程序现在支持两种输入方式：

1. 正常模式
   - 用户输入 title / tags / description

2. Rough idea 模式
   - 用户只输入一句粗略想法
   - 系统会先把这句话改写成更适合检索的 search title 和 search tags


====================
2. 把输入变成语义向量
====================

系统会把用户输入的内容编码成 4 个特征向量：

1. topic
2. format
3. context
4. style

这 4 个向量用来表示：

1. 视频主题是什么
2. 视频形式是什么
3. 视频场景是什么
4. 视频风格是什么


====================
3. 在数据库里找相似视频
====================

数据库里的每条视频也已经提前算好了同样的 4 个向量。

所以程序会：

1. 拿用户输入的 4 个向量
2. 和数据库里的视频做相似度比较
3. 找出一批最相关的视频候选

如果前端勾选了不同的特征按钮，
这些特征的权重也会跟着变化。


====================
4. 选出最终参考视频
====================

程序不是直接取“最像的前几个视频”。

而是：

1. 先保留足够相关的一批视频
2. 再从这批视频里，优先选播放量更高的

这样返回的视频不只是“像”，
而且更有参考价值。


====================
5. 分析参考视频
====================

选出最终参考视频之后，
程序会做两件事：

1. 分析这些视频的 title / tags / description
2. 分析这些视频的 transcript / content / moments

目的是找出：

1. metadata 上可以借鉴什么
2. 内容创作上可以借鉴什么


====================
6. 生成输出结果
====================

最后页面会输出 3 个模块：

1. Video Title, Tags & Description
   - 给用户 title / tags / description 的修改建议

2. Creation Tips
   - 给用户视频内容创作上的建议

3. Matched Videos
   - 展示真正匹配到的参考视频


====================
7. 特殊情况：没有匹配到视频
====================

如果系统没有找到足够相关的参考视频：

1. metadata 模块不会直接断掉
2. 模型会切换成 general AI suggestions 模式
3. 也就是基于用户自己的输入，给更通用的建议

这样即使没有 matched videos，
程序也还能继续工作。


====================
8. 一句话总结
====================

一句话总结：

用户输入自己的视频草稿或 idea，
系统先找相似视频，
再分析这些视频里值得借鉴的地方，
最后输出 metadata 建议、creation tips 和 matched videos。
